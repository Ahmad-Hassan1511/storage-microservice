Keycloak healthcheck notes (CRITICAL — get this wrong and storage-api never starts):
- The official Keycloak 26.2 image is UBI-minimal: NO curl binary exists. CMD-SHELL curl will always fail.
- Health endpoints are served on the MANAGEMENT port 9000, not the application port 8080.
- The --health-enabled=true flag must be passed in the command; it is disabled by default.
- Use a TCP probe to port 9000 instead of curl. The probe below uses bash /dev/tcp:
  ["CMD-SHELL", "exec 3<>/dev/tcp/localhost/9000 && echo -e 'GET /health/ready HTTP/1.0\\r\\n\\r\\n' >&3 && cat <&3 | grep -q '200 OK'"]
  Alternatively, use wget if available: ["CMD-SHELL", "wget -q -O - http://localhost:9000/health/ready | grep -q UP"]
- Expose port 9000 alongside 8080.

Ocelot NuGet package: Ocelot 24.1.0 (stable, runs on .NET 10 via backward compat with net8.0 TFM)

Ocelot routing config structure:
- /storage/v1/{everything} → storage-api:8080  [JWT required — AuthenticationProviderKey: "keycloak"]
- /api/documents/{everything} → documents-api:8080 (returns 502 until Phase 7 — intentional, no auth needed)
- /health → handled directly in Storage.Gateway Program.cs (bypasses Ocelot routing, no auth)

JWT authority URL pattern: http://keycloak:8080/realms/storage-service
(Use the Docker service name "keycloak" not "localhost" — services communicate on the Docker network)

       Note: Ocelot 24.1.0 targets net8.0 but runs fine on .NET 10 runtime via backward compatibility. This is the stable release — use it. Ocelot 25.0.0-beta.2 explicitly targets net10.0 but is in beta; do NOT use beta.
       Note: `Microsoft.AspNetCore.Authentication.JwtBearer` is a framework package in ASP.NET Core — no version needed; it resolves from the shared framework.

    2. Create `backend/src/Storage.Gateway/Program.cs`:
       ```csharp
       using Microsoft.AspNetCore.Authentication.JwtBearer;
       using Ocelot.DependencyInjection;
       using Ocelot.Middleware;

       var builder = WebApplication.CreateBuilder(args);

       builder.Configuration.AddJsonFile("ocelot.json", optional: false, reloadOnChange: true);

       // SETUP-04: JWT validation against Keycloak storage-service realm.
       // The authority URL uses the Docker service name "keycloak" (not localhost) so that
       // the Gateway container can reach Keycloak on the internal Docker network.
       builder.Services
           .AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
           .AddJwtBearer("keycloak", options =>
           {
               options.Authority = builder.Configuration["Keycloak:Authority"];
               options.RequireHttpsMetadata = false;  // Demo/local only — HTTPS required in production
               options.Audience = "storage-api";
               options.TokenValidationParameters.ValidateAudience = true;
           });

       builder.Services.AddOcelot(builder.Configuration);

       var app = builder.Build();

       app.UseAuthentication();

       // Gateway health probe (does not go through Ocelot routing, does not require auth)
       app.MapGet("/health", () => Results.Ok(new { status = "healthy", service = "ocelot-gateway" }))
          .AllowAnonymous();

       await app.UseOcelot();

       app.Run();
       ```
