InitiateUploadAsync steps (in order):
1. Compute payload hash: SHA-256 of `$"{req.CategoryId}|{req.OriginalFileName}|{req.SizeBytes}|{req.OwnerService}"` as hex string. OwnerService is included so that two services with the same category/filename/size but different OwnerService values do not collide on the same IdempotencyKey.
2. Cache key: `$"idempotency:{req.IdempotencyKey}"`
3. Check cache: if hit, compare stored hash to payload hash → IdempotencyConflictError on mismatch, return cached InitiateUploadResponse on match
4. Resolve category via `uow.Categories.GetByIdAsync(req.CategoryId, ct)` → NotFoundError if null
5. Check AllowedOwnerServices: `if (!category.AllowedOwnerServices.Contains(req.OwnerService)) return AccessDeniedError`
6. Create domain file: `DomainFile.Create(caller.TenantId, req.OwnerService, req.CategoryId, req.OriginalFileName, req.MimeType, req.SizeBytes)`
7. Validate via `category.Validate(file)` → PolicyViolationError on failure:
   - Size violation → HttpStatusHint = 413
   - MIME violation → HttpStatusHint = 415
   - Extension violation → HttpStatusHint = 415
8. Compute multipartRequired: `category.IsLargeFile || (category.MultipartThresholdBytes.HasValue && req.SizeBytes > category.MultipartThresholdBytes.Value)`
9. Build storage key: `StorageKey.Create(caller.TenantId, DateOnly.FromDateTime(timeProvider.GetUtcNow().DateTime), file.Id)`
10. Call `storage.GetUploadTargetAsync(putRequest, ct)` — putRequest uses storage key and category's typical content type
11. Save file: `uow.Files.AddAsync(file, ct)` then `uow.SaveChangesAsync(ct)`
12. Publish FileCreatedIntegrationEvent
13. Store idempotency record in cache with TTL of 24 hours
14. Return Result.Success(new InitiateUploadResponse(...))

CompleteUploadAsync steps:
1. Load file: `uow.Files.GetByIdAsync(fileId, caller.TenantId, ct)` → NotFoundError if null or wrong tenant
2. Parse and validate supplied checksum format: `new Checksum(req.ChecksumSha256)` — catches InvalidChecksumException → PolicyViolationError(400)
3. Compare to stored checksum: if file.Checksum is null or !file.Checksum.Equals(supplied) → ChecksumMismatchError
4. Transition: `file.Transition(FileStatus.Scanning)` — let InvalidStatusTransitionException bubble up
5. `uow.SaveChangesAsync(ct)`
6. Map domain event → integration event: create FileUploadedIntegrationEvent from file properties
7. `eventBus.PublishAsync(integrationEvent, ct)`
8. `file.ClearDomainEvents()`
9. Return Result.Success(new CompleteUploadResponse(file.Id, file.Status.ToString()))

CRITICAL anti-patterns to avoid:
- Do NOT use System.IO.File anywhere — use DomainFile alias
- Do NOT publish FileUploadedEvent (Storage.Domain) — publish FileUploadedIntegrationEvent (Storage.Application.Events)
- Do NOT call file.Transition BEFORE SaveChangesAsync
- Do NOT skip AllowedOwnerServices check (it is separate from category.Validate)
- Do NOT omit OwnerService from the payload hash — that would allow cross-service idempotency key collisions
