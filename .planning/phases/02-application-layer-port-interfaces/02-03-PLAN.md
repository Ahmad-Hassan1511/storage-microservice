GREEN phase: Create backend/src/Storage.Application/Services/DownloadService.cs. Use primary constructor syntax.

Constructor parameters: IFileStorageProvider storage, ICacheProvider cache, IUnitOfWork unitOfWork, TimeProvider timeProvider

GetFileAsync steps:
1. Cache key: `$"file:{fileId}"`. Try `cache.GetAsync<GetFileResponse>(cacheKey, ct)`.
2. On cache miss: `uow.Files.GetByIdAsync(fileId, caller.TenantId, ct)` → return NotFoundError if null
3. Gate: if `file.Status != FileStatus.Ready` → return NotFoundError (intentional — architecture §6.5 says return 404 for non-ready)
4. ACL check: scan `file.Permissions` for an entry matching `(caller.PrincipalType, caller.PrincipalId)` with `CanRead=true`. If none found, return AccessDeniedError. Exception: if Visibility is Public (Storage.Domain.Enums.Visibility.Public), skip ACL check.
5. Generate presigned download URL: `storage.GetDownloadTargetAsync(bucket, file.StorageKey.Value, ttl: TimeSpan.FromMinutes(15), ct)` where bucket is derived from the StorageKey or a configured bucket name (use `"files"` as default bucket in this layer; adapters know the real bucket).
6. Build GetFileResponse (include PreviewUrl/ThumbnailUrl only if file.PreviewFileId/ThumbnailFileId are non-null — those require a second GetDownloadTargetAsync call with the preview/thumbnail StorageKey; leave as null for now if not available in the domain model).
7. Cache the response: `cache.SetAsync(cacheKey, response, TimeSpan.FromMinutes(5), ct)`
8. Return Result.Success(response)

GetFileStreamAsync steps (audited proxy path — APP-06):
1. Same cache-first lookup as GetFileAsync steps 1-4
2. Write an AuditEntry: `uow.Audit.AddAsync(new AuditEntry { ... }, ct)` (use the AuditEntry domain entity; fill Action="proxy-download", ActorId=caller.PrincipalId, FileId=fileId, OccurredAt=timeProvider.GetUtcNow().DateTime)
3. `uow.SaveChangesAsync(ct)` (persists audit)
4. Call `storage.OpenReadStreamAsync(bucket, file.StorageKey.Value, range: null, ct)` and return the stream wrapped in Result.Success

CRITICAL anti-patterns to avoid:
- Do NOT check Status after generating presigned URL — check BEFORE (pitfall 3 in research)
- Do NOT query file without TenantId scope (pitfall 4 in research) — GetByIdAsync always receives caller.TenantId
- Return NotFoundError for cross-tenant (not AccessDeniedError) — anti-enumeration
- Do NOT use System.IO.File — use DomainFile alias

For the cache-hit path in tests: when cache returns a non-null GetFileResponse, DownloadService should return it directly without calling uow.Files.GetByIdAsync. Verify this with `uow.Files.DidNotReceive().GetByIdAsync(...)` in the cache-hit test variant if you add one (optional but good practice).
